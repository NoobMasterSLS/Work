===============================================================
Это небольшой свод правил и рекомендаций по использованию программы.
===============================================================

ПЕРЕД ПЕРВЫМ ЗАПУСКОМ (подготовка окружения)
----------------------------------------------------------------

Установка необходимых библиотек (через терминал):
     pip install django openpyxl

Примените миграции для создания базы данных:
     python manage.py migrate

Запустите программу:
     python main.py


ПРАВИЛА (при их нарушении программа может работать некорректно):
----------------------------------------------------------------
1) Файл шаблона template.xlsx должен находиться в той же папке,
   что и исполняемый файл программы (main.py).

2) В любом Excel-файле, который вы загружаете в базу (пункт меню 1),
   обязательно должна присутствовать строка с текстом
   «Информация о сотрудниках» (именно так, с большой буквы,
   без лишних пробелов). По этой строке программа определяет
   начало блока сотрудников.

3) Поля общей информации компании должны располагаться в столбце A
   (название поля) и столбце B (значение поля). Не допускается
   наличие пустых строк между строками с полями до момента
   обнаружения строки «Информация о сотрудниках».

4) Обязательно наличие поля «Название» со значением в столбце B.
   Без него программа не сможет сохранить данные, так как имя
   компании используется как уникальный идентификатор в базе данных.

5) Данные сотрудников должны начинаться после строки
   «Информация о сотрудниках». Если сразу после неё идёт строка
   с заголовками (Фамилия, Имя, Отчество, …), то данные
   сотрудников читаются со следующей строки.
   - Столбец A может содержать порядковый номер сотрудника
     (необязательно, при загрузке игнорируется).
   - Столбцы B–I содержат данные: фамилия, имя, отчество,
     дата рождения, дата трудоустройства, телефон, должность, оклад.
   - Все поля могут быть в любом формате (сохраняются как строки).

6) Не изменяйте структуру шаблона выше строки «Информация о сотрудниках»:
   не удаляйте и не добавляйте строки, не меняйте порядок полей
   общей информации. Это может нарушить чтение/запись данных.

7) При создании нового шаблона через пункт меню 4 программа
   копирует исходный файл template.xlsx. Не удаляйте,
   не переименовывайте и не перемещайте template.xlsx,
   иначе создание новых шаблонов станет невозможным.

8) База данных хранится в файле db.sqlite3. Не редактируйте его
   вручную без необходимости. Для просмотра данных используйте
   пункт меню 3 или Django-админку (см. рекомендации).

================================================================
РЕКОМЕНДАЦИИ:
----------------------------------------------------------------
1) В базу данных сохраняются только данные, введённые в зелёных
   областях шаблона (поля общей информации и строки сотрудников).
   Всё остальное оформление (цвета, объединение ячеек, курсив)
   сохраняется в шаблоне и не влияет на данные.

2) При ручном вводе пути (способ 2) вы можете указать:
   - Полный путь к файлу, например:
     C:\Users\Имя\Desktop\Моя_компания.xlsx
   - Или только папку, например:
     C:\Users\Имя\Desktop
   Если указана папка, программа автоматически создаст в ней
   файл с уникальным именем «Новая_компания.xlsx»,
   «Новая_компания 1.xlsx» и т.д. Если указан полный путь
   к файлу, программа создаст файл с этим именем; если такой
   файл уже существует, она автоматически добавит номер,
   чтобы избежать перезаписи.

3) Для просмотра и редактирования данных через веб-интерфейс
   можно воспользоваться Django-админкой:
   - Создайте суперпользователя: python manage.py createsuperuser
   - Запустите сервер: python manage.py runserver
   - Откройте браузер по адресу http://127.0.0.1:8000/admin
   В админке доступны модели Company и Employee.

4) После каждого обновления кода (например, после pull с GitHub)
   рекомендуется повторно выполнить миграции:
     python manage.py migrate

5) При возникновении ошибок проверьте, что все шаги подготовки
   выполнены, а структура Excel-файла соответствует правилам.
